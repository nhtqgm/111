# 股票K线人工预测对比

这是一个从零开始的 MVP，用东方财富历史 K 线作为真实数据源，支持对申万宏源 `000166` 等 A 股进行人工预测线录入和对比。

## 已实现

- 东方财富历史 K 线接口接入
- 日K / 周K / 月K 切换
- 5 / 10 / 20 个周期同屏预置和展示
- 预测起点默认当前日期，也可以选择历史日期或未来日期
- 系统从预测起点当天开始生成最长 20 个目标周期
- 目标周期如果能匹配东方财富真实K线日期，会显示真实价并参与误差计算
- 人工录入预测收盘价
- 同一目标日期在 5 / 10 / 20 中重复出现时，输入一个值会自动回填其他空列
- 真实 K 线、真实收盘线、5/10/20 三条人工预测线同图展示
- 已有真实值的周期自动计算误差、MAE、MAPE
- 预测值按 `股票 + K线级别 + 预测起点 + 预测长度` 自动保存到本机缓存
- 自动恢复上次使用的股票、K线级别、预测起点、预测长度和预测表
- 支持将当前预测方案导出为 JSON 文件，也支持从 JSON 文件导入恢复，作为备份/迁移能力

## 运行

Windows 上可以直接双击：

```text
start-gupiao.cmd
```

也可以手动运行：

```bash
npm install
npm run dev
```

打开：

```text
http://localhost:5173/
```

## 预测数据保存

录入的预测值会自动保存到当前电脑的本地缓存。刷新页面、关闭浏览器、重启电脑后会自动恢复上次工作状态。

自动缓存包含：

```text
上次使用的股票代码
日K / 周K / 月K
5 / 10 / 20 三组预测
预测起点
每个目标周期的预测收盘价
备注
```

每台电脑的缓存是独立的。别人拿到软件后，在他自己的电脑上录入的数据会自动保存在他自己的电脑上。

如果要把你电脑上的预测方案转移给别人，才需要使用右侧预测面板里的：

```text
导出
导入
```

导出的 JSON 文件会包含：

```text
股票代码
K线级别
预测起点
预测长度
每个目标周期的预测收盘价
备注
```

## 东方财富参数

当前通过 Vite 本地代理请求东方财富：

```text
/eastmoney/api/qt/stock/kline/get
```

关键参数：

```text
secid=0.000166
klt=101 日K
klt=102 周K
klt=103 月K
fqt=1 前复权
```

深圳市场使用 `0.xxxxxx`，上海市场使用 `1.xxxxxx`。当前代码按股票代码首位自动判断。
