# 未归属历史重命名设计

## 目标

允许用户在预测复盘弹窗中重命名“未归属历史”的显示名称，同时保持这些记录仍属于内部 legacy 分组，不修改快照的 `planId`、快照 ID 或复盘筛选语义。

## 方案

- 在复盘范围筛选旁增加一个紧凑的“重命名”按钮，仅在当前股票和周期存在未归属历史记录时显示。
- 点击后沿用现有方案重命名交互，通过输入框设置新名称。
- 空名称不保存；名称会去除首尾空格，并限制为 30 个字符。
- 新名称同时用于筛选选项和复盘记录的“预测来源”显示。
- 重命名只影响当前 `股票代码 + 日K/周K/月K` 工作区。

## 数据与缓存

- 使用独立缓存 key：`prediction-ma:replay-legacy-label:<股票代码>:<周期>:v1`。
- 缓存值保存为普通字符串；缺失或无效时回退为“未归属历史”。
- 保存后调用现有 Electron 全量存储同步，因此网页刷新、EXE 重启、全量导出和全量导入都能保留名称。
- 不修改任何历史快照内容，避免破坏已有去重、复盘匹配和方案隔离逻辑。

## 状态流

1. 股票或周期数据加载完成后，读取对应工作区的 legacy 显示名称。
2. 用户重命名后，先更新当前页面状态，再写入本地缓存并同步 Electron 存储。
3. 切换股票或周期时，重新读取目标工作区名称，不复用上一个工作区的名称。
4. 导入全量备份后，现有页面重载流程从导入后的缓存恢复名称。

## 验证

- 单元测试覆盖默认值、名称规范化、按股票和周期隔离、保存及 Electron 同步。
- 浏览器验证重命名后筛选选项和预测来源同步变化，刷新后仍保留。
- 运行 `npm run test:storage`、`npm run verify:ma`、`npm run build` 和 `git diff --check`。

## 非目标

- 不把未归属历史转换成正式预测方案。
- 不修改历史快照归属或批量重写历史数据。
- 不增加全局统一名称设置。
