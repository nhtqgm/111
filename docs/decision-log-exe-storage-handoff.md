# EXE 数据接力决策记录

## 背景

EXE 联网时加载 GitHub Pages，离线时加载内置 `file://` 页面。两个来源的
`localStorage` 相互隔离，导致联网更新 UI 后需要手工导入数据。

## 决策

- Electron 主进程在 `app.getPath('userData')/app-data` 下维护 `app-cache-v1.json`。
- 只允许保存 `prediction-ma:*` 和 `prediction-ma40:*` 两类键。
- 页面挂载 React 之前，先通过 preload IPC 合并当前来源数据与固定快照。
- 手动保存、30 秒自动保存和 K 线缓存更新后，把完整应用快照同步到主进程。
- 第一次运行新版 EXE 时，隐藏加载内置页面并迁移旧 `file://` 数据，再加载在线 UI。
- 写入使用临时文件替换，并保留最近 5 份备份。

## 拒绝的方案

- 只使用在线页面：无法保证离线运行。
- 只使用内置页面：失去前端在线更新能力。
- 仅在 UI 更新后提示导入：仍依赖用户操作，不能解决数据隔离根因。

## 风险控制

- 在线空表不能覆盖已有的有效预测值。
- 带 `updatedAt` 的工作区和 K 线缓存保留较新版本。
- 数据恢复失败时不挂载业务页面，防止空表自动保存。
- IPC 限制为正式 GitHub Pages 和内置 `file://` 页面调用。

## 回滚

可以回滚 Electron 和前端代码，但必须保留用户目录中的 `app-cache-v1.json` 与
`backups`。回滚后可通过新版全量导出文件恢复到旧版本。
